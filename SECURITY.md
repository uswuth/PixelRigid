# Security Policy

## Supported Versions

PixelRigid is an actively developed project. Only the latest release receives security fixes and bug fixes.

| Version | Supported |
| -------- | --------- |
| Latest Release | ✅ |
| Previous Releases | ❌ |
| Development Branch | ⚠️ Best effort |
| Older Releases | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability in PixelRigid, please report it privately instead of opening a public issue.

### How to Report

Use one of the following methods:

- **GitHub Security Advisories** (preferred)
- **GitHub Private Vulnerability Report**
- Contact the maintainer directly if instructed in the repository.

Please include:

- A detailed description of the vulnerability
- Steps to reproduce
- Windows version
- PixelRigid version
- Screenshots or logs (if applicable)
- Proof-of-concept code (if available)

---

## Response Timeline

The goal is to respond within:

- **Acknowledgement:** Within 72 hours
- **Initial Assessment:** Within 7 days
- **Status Updates:** As progress is made
- **Security Fix:** Released as soon as a verified fix is available

These timeframes are goals and may vary depending on the severity and complexity of the issue.

---

## Scope

Examples of security issues include:

- Privilege escalation
- Arbitrary code execution
- DLL hijacking
- Process injection vulnerabilities
- Unsafe Windows API usage
- Command injection
- Unsigned binary replacement
- Memory corruption
- Any vulnerability that could compromise a user's system

---

## Out of Scope

The following are generally not considered security vulnerabilities:

- UI bugs
- Feature requests
- Compatibility issues
- Cosmetic issues
- Performance problems
- Crashes without security impact

---

## Disclosure Policy

Please allow time for the issue to be investigated and fixed before making it public.

Once a fix is available, the vulnerability may be disclosed through the project's release notes or GitHub Security Advisories.

Thank you for helping keep PixelRigid secure.
